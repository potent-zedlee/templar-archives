## 로그 작성 규칙

### 세션 로그 형식

```markdown
## YYYY-MM-DD (세션 N) - 제목

### 작업 내용
- 주요 작업 1
- 주요 작업 2

### 핵심 파일
- `path/to/file.ts` (신규/수정, N줄)

### 완료 기준 달성
- [ ] 체크리스트

### 다음 작업
- [ ] 예정 작업
```

### Phase 로그 형식

```markdown
## Phase N: 제목
- **주요 기능**: 설명
- **파일**: N개 파일, M줄 추가
- **마이그레이션**: `migration_name.sql`
- **커밋**: hash
- **완성도**: YYYY-MM-DD 완료
```
