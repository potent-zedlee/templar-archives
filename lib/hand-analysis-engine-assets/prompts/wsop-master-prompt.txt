[화면 구조]
- 방송 스타일: ESPN 스포츠 중계 스타일
- 플레이어 박스: 테이블 주변에 직사각형 박스 배치
- 플레이어 정보 (각 박스):
  - 플레이어 이름 (예: "PHIL HELLMUTH")
  - 스택 크기 (칩 단위 또는 BB 배수, 예: "150,000" 또는 "75 BB")
  - 홀 카드 (공개 시)
  - 추가 통계 (VPIP, PFR 등) - 선택적으로 표시

- 토너먼트 정보: 화면 상단 중앙
  - WSOP 로고
  - 이벤트 이름 (예: "Main Event Day 5")
  - 블라인드 레벨 (예: "Level 28: 50K/100K/10K")
  - 남은 플레이어 수 (예: "18 Players Remaining")

- 커뮤니티 카드: 화면 중앙 또는 하단
  - 5장의 카드, ESPN 스타일 그래픽

- POT 크기: 커뮤니티 카드 근처
  - 칩 단위 표시 (예: "450,000")

[UI 특성]
- 배경: 녹색 펠트 테이블, 클래식 포커 룩
- 폰트: 깔끔한 sans-serif, 흰색/노란색 텍스트
- 애니메이션: 중간 수준 (ESPN 방송 스타일)
- 해설: 2명의 해설자 (주 해설자 + 포커 프로)
  - 매우 상세한 해설 제공
  - 플레이어 전략, 핸드 레인지 분석

[중요 주의사항]
1. **핸드 재생(Hand Replay) 감지**
   - 일부 핸드는 나중에 다시 보여줌 (리플레이)
   - 타임스탬프가 중복되면 스킵
   - 리플레이는 "EARLIER" 또는 "REPLAY" 텍스트로 표시

2. **블라인드 레벨 추적**
   - 핸드마다 블라인드가 다를 수 있음 (토너먼트 진행 중)
   - 화면 상단의 블라인드 정보를 각 핸드마다 기록

3. **BB 배수 표시**
   - 일부 스트림은 스택을 BB 배수로 표시 (예: "75 BB")
   - BB 단위로 변환: 75 BB × 100K = 7,500,000

4. **통계 정보 무시**
   - VPIP, PFR 등 추가 통계는 핸드 히스토리에 불필요
   - 플레이어 이름, 스택, 카드, 액션만 추출

[핸드 시작 감지 - 5가지 신호]

**Signal 1: 딜러 버튼 이동**
- WSOP: 테이블 위 "DEALER" 또는 "D" 버튼

**Signal 2: 블라인드 포스팅**
- SB와 BB 강제 베팅
- 화면 상단에 블라인드 레벨 표시

**Signal 3: 홀 카드 딜링**
- 각 플레이어에게 2장의 카드 딜링
- ESPN 스타일 카드 등장 효과

**Signal 4: POT 초기화**
- 이전 POT 리셋, 새 POT = SB + BB + Ante

**Signal 5: 플레이어 스택 업데이트**
- 이전 핸드 결과 반영

[토너먼트 특수 규칙]

1. **Ante (앤티)**
   - 모든 플레이어가 강제 베팅
   - POT 초기값 = SB + BB + (Ante × Player Count)

2. **All-In 보호**
   - 스택이 부족한 플레이어는 자동으로 올인
   - "All-in and Called" 상황 빈번

3. **Bubble (버블)**
   - 상금권 진입 직전
   - 해설자가 "We're on the bubble" 언급
